# 📋 מערכת ניהול משתמשים - Ness Assignment

פרויקט זה מציג פתרון מלא (Full-stack) לניהול משתמשים בארגון, תוך דגש על חוויית משתמש רציפה (Inline Editing), שלמות נתונים והפרדה בין שכבות המידע.

---

## 🏗️ תהליך הפיתוח והארכיטקטורה

הפיתוח התמקד ביצירת מערכת יציבה המאפשרת ניהול נתונים מורכבים בצורה פשוטה למשתמש הקצה. להלן פירוט שלבי הפיתוח המרכזיים:

### 1. שכבת הנתונים וניהול ישויות (Backend Logic)
*   **מיפוי גמיש (Fluent Mapping):** במקום להשתמש בהגדרות בסיסיות, השתמשתי ב-Fluent NHibernate כדי להגדיר קשרים מורכבים בין טבלאות (כמו קשר בין משתמש לתפקידו ורמת הארגון שלו), מה שמאפשר שליפת נתונים יעילה (Lazy Loading vs. Fetching).
*   **הפרדת ישויות (DTO Pattern):** פיתחתי מודלים נפרדים לתצוגה (Models) וישויות לבסיס הנתונים (Entities). המעבר ביניהם מתבצע אוטומטית בעזרת **AutoMapper**, מה ששומר על אבטחת המידע (אי-חשיפת סיסמאות ו-Salts ל-Client).
*   **ניהול טרנזקציות:** כל פעולת כתיבה (הוספה או עדכון) עוטפת ב-Transaction של NHibernate כדי להבטיח שבמקרה של שגיאה, בסיס הנתונים לא ייפגע ויישאר עקבי.

### 2. ממשק המשתמש (Frontend Logic)
*   **עריכה בשורה (Inline Editing):** פיתחתי מנגנון המאפשר להפוך כל שורה בטבלה למצב עריכה בלחיצת כפתור, ללא צורך במעבר דפים או פתיחת מודאלים מורכבים, מה שמשפר משמעותית את מהירות העבודה.
*   **סינון וחיפוש דינמי:** המערכת מבצעת סינון בזמן אמת (Client-side filtering) על המידע שנמשך מהשרת, תוך שילוב של חיפוש טקסט חופשי (שם משתמש) ובחירה מתוך רשימת תפקידים דינמית.

### 3. מנגנון ולידציה כפול (Validation Strategy)
*   **ולידציה צד-לקוח:** שימוש ב-Regex מתקדם ב-Angular לאימות פורמט טלפון ישראלי וכתובות אימייל. הכפתור "שמור" ננעל אוטומטית במידה והנתונים אינם תקינים, כדי למנוע שליחת בקשות שגויות לשרת.
*   **ולידציה צד-שרת:** יצירת פונקציות ייעודיות לבדיקת "תקינות נתונים" (Well-formed data). השרת מסנן משתמשים עם נתונים חסרים או לא עקביים ומחזיר רק נתונים איכותיים לאפליקציה.

### 4. פתרון בעיות ואופטימיזציה
*   **טיפול ב-Transient Instances:** פתרתי בעיות קישוריות ב-ORM על ידי שימוש ב-`session.Load` כדי לקשר ישויות קיימות (כמו תפקידים) למשתמשים חדשים מבלי ליצור כפילויות בבסיס הנתונים.
*   **סנכרון תצוגה:** המערכת מעדכנת את ה-UI מיד עם קבלת אישור מהשרת (Success response), ללא צורך ברענון הדף כולו, תוך שימוש ב-Spread Operator לעדכון אופטימי של הרשימה.

---

## 🚀 תכונות עיקריות
*   **ניהול תפקידים:** תמיכה בטבלאות Lookup דינמיות.
*   **אבטחה:** עבודה עם Salt וסיסמאות מוצפנות (בצד השרת).
*   **עיצוב רספונסיבי:** התאמה למסכים שונים עם CSS מותאם אישית.
*   **ממשק Swagger:** תיעוד מלא של ה-API לבדיקה מהירה של ה-Endpoints.

---

## 🛠️ הנחיות להפעלה
1.  הגדרת מחרוזת התקשרות (Connection String) בקובץ `appsettings.json`.
2.  הרצת ה-Backend ולוודא שה-Session Factory מאותחל בהצלחה.
3.  הרצת ה-Angular Client (`npm start`) וביצוע `npm install` במידת הצורך.
